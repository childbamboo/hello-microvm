# 開発要件

## 1. プロジェクト概要

microVM を使ったリモート AI 開発デモ。
チャットで機能要望を出すと、AI エージェントが microVM 上で自動開発し、完成物を Web プレビューできるシステム。

### 位置づけ

- microVM（e2b）でどこまでできるかの **PoC**
- PoC であるが、最終的には **顧客への提案資料** としても活用する

## 2. 技術スタック

| レイヤー | 技術 |
|---|---|
| フロントエンド | Next.js |
| microVM | e2b（クラウド） |
| AI エージェント | LLM（詳細は別途選定） |

## 3. 機能要件

### 3.1 チャット UI

- ユーザーがチャット形式で機能要望を入力できる
- AI エージェントが要望を解釈し、開発タスクに変換する

### 3.2 microVM の起動と自動開発

- チャットでの要望受信をトリガーに microVM（e2b）を起動する
- AI エージェントが microVM 内でコーディング・ビルドを自動実行する
- 対象アプリはモックアップ用途の Web アプリ全般
  - バックエンド・データベースは簡易なもので構成する

### 3.3 作業状況の表示

- 開発中、フロントエンドで microVM の作業状況をリアルタイム表示する
- 表示形式:
  - **ターミナル出力**: microVM 内のターミナル出力をストリーミング表示
  - **タスク進捗**: タスク単位の進捗やログメッセージを表示

### 3.4 Web 公開とプレビュー

- 開発完了後、成果物を microVM 上でホスティングし公開 URL を発行する（e2b の機能を活用）
- フロントエンドからプレビューとして参照可能にする（iframe 埋め込み or リンク）

## 4. 画面構成（概要）

```
+-----------------------------------------------+
|  ヘッダー                                       |
+-------------------+---------------------------+
|                   |                           |
|   チャット         |   作業状況パネル            |
|   エリア           |   - ターミナル出力          |
|                   |   - タスク進捗              |
|                   |                           |
+-------------------+---------------------------+
|   プレビューエリア（開発完了後に表示）              |
+-----------------------------------------------+
```

## 5. 今後の確認事項

| # | 項目 | 内容 |
|---|---|---|
| 1 | LLM の選定 | microVM 内で動かす AI エージェントに使う LLM（Claude / GPT 等）と呼び出し方式（API / CLI） |
| 2 | 認証 | PoC 段階で認証機能は必要か（デモ用途なら不要の可能性あり） |
| 3 | 同時利用 | 複数ユーザーが同時にセッションを持てる必要があるか |
| 4 | microVM のライフサイクル | セッション終了後の microVM の扱い（即破棄 / 一定期間保持） |
| 5 | 公開 URL の有効期間 | プレビュー URL をどの程度の期間維持するか |
| 6 | 開発対象の制約 | テンプレートを用意するか、フリーフォームで開発させるか |
